# Results — SNG-BATCH-5 (Skill-tree tradeoffs)
Date: 2026-07-10 · Shipped v1.6.9 · Suite green (442 checks) + parse_probe. Verified live.

## Phase 1 — Soft class cost: SHIPPED
- **engine/skilltree.js** — `homeClassOf(character)` (origin → home power system: harmonic/radiant own; valley → valley_craft), `isCrossClass(ability, character)` (true when the ability's class system ≠ home; learned/discovery never count; **precursor is never home → always cross**), `skillPointCost(ability, character, skillCapacity)` (1 normally, ×`crossClass.costMultiplier` = 2 cross-class).
- **engine/progression.js** — `learnAbility` and `rankUpAbility` charge the multiplier. Affordability is checked **after** all eligibility (tradition / level-req / attribute-gate / capacity), so a forbidden or under-level ability still reports its real reason, not a price. No hard rank ceiling — you *can* master a secondary class, it just costs double. Stacks with the existing cross-training +1 levelReq.
- **UI** — the doubled cost shows in the learn picker (`· 2 pts`, and a `🔒 need 2 pts` lock when unaffordable) and on the rank-up button (`▲×2`, tooltip "cross-class"). The rank-up highlight names the point spend.
- Smoke: home cost 1 / cross cost 2; learn + rank both blocked at 1 point and succeed at 2; precursor always cross; eligibility precedes affordability.

## Phase 2 — Branch forks: SHIPPED
- **engine/skilltree.js** — `forkFor`, `forkPaths`, `forkPending`, `chosenFork`, `setFork` (permanent once set), and `rankExpression(character, ability, level, branchForks)` — the single fork-aware accessor that returns the chosen path's name/grants/cannot in place of the linear tree entry at/above the fork rank.
- **Flow** — ranking a forking ability (prism_sight / sonic_resonance / greenlore / quiet_step / latticespeak, each at rank 3) TO its `atRank` opens a **choose-a-path modal** (`renderForkModal`, shared by the play sheet and the character screen) with a confirm; picking one calls `setFork` (the other locks permanently) then ranks. `character.forkChoices` persists the choice; migrated on load.
- **Everywhere the rank shows** the chosen path replaces the linear expression: character sheet (⑂ marker + name), rank-up highlight ("specialized"), GM ability law (`abilitiesForGM` now fork-aware → the GM narrates the specialized form), and the **skill graph** (⑂ badge on forking nodes, gold when chosen; node detail shows "specialized as X — Y locked forever").
- Smoke: fork pends only at atRank while unchosen; choosing locks permanently and refuses re-choice; chosen path replaces the rank expression with its grants/cannot; below atRank and unchosen fall back to linear; GM sees the specialized name.
