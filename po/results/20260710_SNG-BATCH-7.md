## Phase 1 — Player identity + per-character play-style (seed-from-aggregate): SHIPPED (v1.7.3)
- **Style moved profile → character.** A character now earns its OWN `tendencies`/`aptitudes`/`actionCount` from how IT is played (Cellaceron gains strategist, Usnea doesn't). `playerprofile.js` functions are generic over a style-holder; every play-site call (`updateProfile`, `aptitudeMods`, `profileInsight`) passes the active character. `sense.js`/resolution untouched — they already take `aptitudeMods` as a value; the CALLER sources it from the character.
- **Profile is now identity only** (`playerKey`/`displayName`/`charactersPlayed`, schemaVersion 2) — exactly what Phase 2 needs.
- **SNG-022 seed step (v3):** an existing character with no style inherits the player's CURRENT aggregate profile once (nothing earned is lost), then diverges; idempotent (reconcileVersion gate); never clobbers a character that already has style; login note "your characters now carry their own play-style". `topReconcileVersion()` lets a freshly-built character stamp born-current so it never seeds.
- **Identity (path-a, no auth):** `hasChosenPlayer()`/`listPlayers()`; a "Who's playing?" picker appears when a device knows >1 player and hasn't chosen; roster shows the active player + a "switch" link (Tether Maven-select pattern). Repo-write PAT stays device-config (path-a boundary — blast radius = Singularity-repo-write only; flagged).
- Live-verified: per-character accrual; seed-from-aggregate w/ login note; picker lists local players; born-current chars don't seed.

## Phase 2 — Cross-device persistence (load-latest, guarded): SHIPPED (v1.7.4)
- **`resolveSaveConflict(local, remote)` — PURE, 13 unit checks — NEWER WINS** (updatedAt primary, rev tiebreak). A both-advanced conflict is flagged ONLY when both sides moved past a common `syncedAt`; the loser is returned for preservation, never dropped.
- **STALE-LOCAL-OVERWRITE GUARD (the named failure mode) fires in BOTH directions:** `saveCharacter` stamps `updatedAt`+`rev`; `adoptRemoteCharacter` writes a pulled copy preserving its timestamps + marks `syncedAt`; `pushCharacterGuarded` re-reads remote and REFUSES to overwrite a fresher one (push side); opening a character PULLS latest BEFORE migrate/play so a stale local is replaced, never re-stamped as fresh (pull side). A conflict preserves the losing copy under a recovery key + surfaces "Loaded your latest from another device" / recovery note.
- Clock-skew note (path-a boundary): `updatedAt` uses each device's clock — fine for Erik's own family devices; a version-vector would be the public-multi-user upgrade.
- Live-verified: resolver both-directions + conflict; adopt preserves remote timestamps; push-guard present.

## Phase 3 — Inventory + quest robustness (apply the resolver): SHIPPED (v1.7.5)
- **ROOT FIX — the SNG-019 primitive wired into BOTH quests.js and inventory.js.** Extracted `normName`/`namesMatch`/`resolveByName` into dependency-free **`engine/namematch.js`** (avoids the codex↔quests import cycle); codex re-exports namesMatch.
- **Quests:** `resolveQuest` (id → slug-title → title/alias fuzzy). A progress/complete op on a drifted title updates the RIGHT quest; an unresolvable op **surfaces a note, never silently drops**; a `start` that resolves to an existing quest doesn't fork a dupe; giver/location resolve to codex entityIds (a quest about Teva links her node). `dedupeQuests` repairs pre-resolver fragmented saves (idempotent; resolved-status wins; progress unioned).
- **Inventory:** `resolveInventoryItem` (id → name/custom/alias → fuzzy). `addItem` stacks phrasing variants onto one stack + records the drift as an alias + late catalog re-link. `dedupeInventory` collapses forked stacks summing qty (idempotent).
- **Reconcile back-tag (character step v4):** both dedupes run on load (silent repair — the log/panel just render correctly).
- **UI:** a full **Quest Log** (active/completed/failed + progress, giver's codex node linked ◈) reachable from the play screen; quest detail links to the giver's codex node. Inventory panel already existed. *Depth of management UI = Erik's preview-leg to tune.*
- **Design choice — conservative matching:** containment, not stemming, so "Find the Lost Ring" never eats "Find the Lost Dog". Stemming drifts ("clearing" vs "clear") surface a note rather than a silent drop; the player-repair path + dedupe catch stragglers. This honors the spec's primary guarantee (NEVER silently drop) without risking false merges.
- Live-verified: drifted-title progress → right quest; unresolvable → surfaced; giver → teva node; 3 waterskin phrasings → one stack of 3; dedupe collapses fragments keeping resolved status.
