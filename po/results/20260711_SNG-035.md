## Phase 1 — the pipeline (`engine/art.js`, pure up to the URL)
One path for every image: **assemble a prompt → THE FLOORS → endpoint URL → persist-once on the record → display + gallery.**
- **THE FLOORS FOR IMAGES** (`sanitizeImagePrompt`) — the same non-negotiable the text generator enforces (`generate.js enforceFloors`), applied to pixels:
  - **Sexual content stripped at ANY ceiling.**
  - **Gore/graphic-violence stripped below an R ceiling** (kept only at R/R+).
  - **A MINOR subject is forced child-safe** (romance, sexual, and graphic-violence terms scrubbed; a wholesome tone imposed) **at ANY ceiling** — a hard scrub, never a softening. Minor-detection reuses the `_gen.romanceEligible=false` stamp + age/descriptor markers.
  - Every prompt carries an absolute safety tail (no nudity, tasteful, original character not a real person, no copyrighted characters, no text/watermark).
- **Prompt assembly** — `characterPromptSeed` (appearance + origin + gear + current arc), `assembleImagePrompt` (character/npc/location/item/moment), `imageURLFor` (per-kind sizing, stable seed).
- **Persist-once (born-with-image)** — `ensureImage` assembles the prompt exactly once, stores the URL on the record (`character.portrait`, `npc/location.image`), and thereafter returns the stored URL — the image rides the save/sync like any other field, never re-assembled. Same discipline as the BATCH-9 generate store.
- **Gallery** — `ensureGallery` / `addGalleryImage` (dedupe by url, newest-first, capped 48).
- **Toggle + cost-aware** — the existing `artMode` setting (`off`/`static`/`generate`); `imagesEnabled()` is the on-switch. Default `static` (generation off) — Erik flips to `generate`. Pollinations is free/keyless so there's no metered-account cost concern. The existing `locationImage`/`sceneImage`/`itemImage` helpers were rerouted through the floors for defense-in-depth.

## Phase 2 — consumers (all ride the ONE pipeline)
- **CHARACTER PORTRAITS** — the headline. Minted at creation (`finish`), re-minted on a milestone (every 2 levels → a new seed so the image visibly grows), shown on the character sheet with **↻ New portrait** / **✦ Generate portrait** controls.
- **NPC + LOCATION born-with-image** — in `handleGenerateRequests`, a freshly-generated NPC/location is stamped with its image at mint (persists in the generated store, routed through the floors via its `_gen` minor flag). **This IS SNG-046 Layer 3** — a discovered/generated place is born with both its data and its image.
- **MOMENT ART** — the GM emits an `imagePrompt` on a beat worth a picture (a discovery, a gambit's climax, a Tier-IV/precursor use, a world-effect, a striking first sight); the engine renders it within the player's ceiling, clamped ~1/scene, shown inline in the beat and saved to the gallery.
- **THE SAGA GALLERY** — a screen listing every image the character has accrued (portraits, born-with-image people/places, moment art), newest-first (ties SNG-023).

## FLOORS APPLY TO IMAGES (non-negotiable — held)
Every image prompt — portrait, NPC, location, scene, item, moment — routes through `sanitizeImagePrompt` before it reaches the endpoint. No image sexualizes a minor (a hard scrub at any ceiling) and no image exceeds the viewing player's content ceiling. Verified live: `sanitizeImagePrompt("a seductive nude teenager", {ratingLevel: R+, isMinor: true})` → all sexual/minor terms gone, "child, age-appropriate, wholesome, non-sexual" imposed.

## Guardrails held
Extended the settled Pollinations endpoint (did NOT re-decide or swap providers). Floors on images = the text floors applied to pixels (minor-protection absolute, ceiling-respecting). Persist-once (born-with-image; never regenerate) — same discipline as the generate store; cost-aware + toggleable (default off). Content-safe (original character/environment art, no IP, no real people). Additive (`character.portrait`, `character.gallery`, record `.image`, GM `imagePrompt` op) — backward-safe; art off = clean no-op, solo/existing saves unchanged. Never halts a turn (every art path is try/caught and returns null on failure). Design law 1 (engine owns prompt-assembly + floors + persist; the GM only flags a beat worth art). Never touched the ErikIAm pipeline.
