# Results — SNG-041 (Absolute world dating — one world, one clock)
Date: 2026-07-11 · Shipped v1.8.0 · Suite green at every sub-phase (635 checks, +22 over BATCH-9 Phase 1) + parse_probe. Live-verified (clean origin). Status: **shipped, awaiting Erik preview-leg before complete_pending_review**. Only Aevi closes.

**The gate before BATCH-9 Phase 2.** Root bug: `newClock(startDay=1)` gave every character its own Day-1 origin, so "Day 8" (this character) and "Day 11" (the Ent) were private counts with no shared reference — a cross-character event carried one frame's day into another's and couldn't reconcile. SNG-041 establishes the shared absolute so cross-character events are born reconcilable. Built as three green sub-phases; the per-character clock is preserved as the LOCAL play-paced frame — nothing ripped out.

## 41a — Shared world epoch + absolute world-day (foundation)
- `engine/worldtime.js` adds the shared absolute, additive (per-character clocks untouched):
  - A **fixed world epoch** (a constant, never `Date.now()` — a per-device init would give each device a different origin and break "one clock"); `getWorldEpoch`/`setWorldEpoch` with a synced/local override hook for future tuning.
  - `absoluteWorldDay(nowMs, epoch)` — THE reconciliation key: identical on every device, real-time; the far world ages whether or not anyone plays. `worldDate` (day+season+label). `worldDayAt(ISO/ms)` maps a real-time stamp (a ledger event's `.at`) to the absolute, so the same event dates the same on every viewer's calendar (unknown → null; derives-never-fabricates). `relativeWorldDays` ("N days ago" from the viewer's now).
  - All pure + injectable (`nowMs`/`epoch` params) → deterministic tests.
- Green: two characters at different journey-days read the SAME absolute world-day at one real moment.

## 41b — Stamp events/news on the absolute + cross-character reconcile
- **worldtick news** carries the absolute `worldDay` alongside the local journey-day. **New ledger events + deeds** stamp `worldDay` at write time.
- **`syncSharedWorld` reconciliation (the fix):** another character's event dates by the shared absolute — `worldDayAt(e.at)` (or its own `worldDay` stamp) — so their timeline and yours share ONE calendar.
- **`newsForGM`** dates on the shared world-day when known, falls back to the local journey-day for pre-SNG-041 items (no fabricated dates).
- Green: world-tick news is absolute-stamped; a cross-character event resolves to one shared world-day for every viewer; legacy items render gracefully on the journey-day.

## 41c — The hybrid two-clock + GM contract + display
- **The hybrid (Erik's resolved anchor):** the far/ambient world runs on the shared **real-time** absolute (cross-character stamps, propagating events); the active local frame (current area + active quests) is **play-paced** (the per-character clock — it waits for the player). Both shown to the player: the local journey clock + a `· world-day N` chip.
- **GM contract:** the GM is given the shared world calendar and told to **reference dates only as given — never invent a bare day-number** (the engine owns the calendar).
- **Consequence-cross tag:** `ledgerEvents.impactsLocal` marks a distant event that would materially reach ANOTHER character's area or active quest; it crosses the far-world/local boundary and surfaces prominently ("This reaches your area —") on return. The Phase-2 GENERATION of such events stays out of scope — SNG-041 provides the boundary + the tag; Phase 2 fills it.
- Green: the GM context surfaces the world calendar + the references-not-invents rule (backward-safe when absent); a boundary-crossing item renders with its crossing phrasing on the shared date; the contract fields shipped.
