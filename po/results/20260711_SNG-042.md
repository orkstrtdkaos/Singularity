# Results — SNG-042 (Legends & Villains — the world's great figures)

Date: 2026-07-11 · Shipped v1.8.11 · 734 smoke checks (+11) + parse_probe green · legends verified in-browser. Status: **shipped, awaiting Erik preview-legs before complete_pending_review**. Only Aevi closes.

**The world feels large when greatness passes through it.** Rides BATCH-9 (generate + weight/recurrence + codex). New pure module `engine/legends.js`; the Aevi-authored anchors (`legends.json`) now load and deploy.

## 1. Power-tier spectrum
`LEGEND_TIER_WEIGHT` (legendary 50 → epic 45 → regional 16 → notable 10 → riffraff 3) — `tierBirthWeight` makes a legendary figure **born heavy**, so the BATCH-9 weight system keeps it real + recurring by default (the measure others are held against). `tierForArc(level)` scales the tier to the character's arc — riffraff early, regional mid, legendary earned — so no epic villain shows for a tavern scuffle. On both hero and villain alignment.

## 2. The roster
`loadLegends(legends.json)` normalizes the **authored anchors** (3 heroes — Kesh/witness, Iselde/advice, Alder/rescue — + **Halvex** the epic villain + **Grael** regional) into **NPC-shaped, high-weight canon** carrying a `legend` tag (tier, alignment, presencePattern, signature, threading). Hydrated into `CONTENT.npcs` at content-load so **SNG-019 resolves them by name and the GM reuses them, never reinvents**. Mid/low tiers generate in-grain via the BATCH-9 path at the requested tier (`generate.js` now stamps `_gen.legendTier`/`alignment`).

## 3. Dramatic-beat deployment (the novel system — "legend surfacing")
`legendSurfacing(...)` is **RARE + governed**:
- **cooldown** (`minGapDays`, default 6 world-days since the last deploy) so greatness stays special;
- a **rarity roll** so even a qualifying beat mostly passes quietly;
- **power-appropriate**: the tier scales to the character's arc;
- **alignment-matched**: heroic beats (`doomed_rescue` / `witness_power` / `passing_advice`) surface heroes; `villain_escalation` surfaces villains — an authored anchor whose `presencePattern` names the beat is preferred (arc-appropriate, may exceed by one tier for a glimpse of the legendary), else a **generate-at-tier** is signalled.
- `legendDeploymentForGM(...)` gives a **rating-aware** GM directive per beat (doomed rescue / witnessed set-piece / passing advice / villain shadow) that keeps the figure **witnessed and earned — never a deus-ex-machina that steals the player's agency**.

Wired in `app.js`: `detectLegendBeat` fires only on a real set-piece/peril (a lethal encounter with the character in genuine danger → doomed_rescue; a challenge/duel → witness_power); `maybeLegendDetail` runs the governed surfacing (governor state on the character) and feeds `legendDetail` to the GM; `gm.js` consumes it as a world block.

## 4. Threading via recurrence
Legends are high-weight reusable canon: once deployed and met they enter the npc registry + codex and recur naturally (the hero who held a line is later fought beside; a rescuer remembers you). With Phase-2 offscreen advancement + Phase-3 promotion, a legend moves through and can enter shared canon.

## 5. Governors
Rare, earned deployment (cooldown + rarity); power-appropriate (tier scales to arc); villain tier scales to the character's arc; **rating-aware** (the directive respects the player's ceiling — ties the BATCH-9 rating-lens). Engine detects the beat + selects the figure; the GM narrates.

## Guardrails held
Engine detects the beat + selects the figure; GM narrates. Rare/governed — greatness isn't cheapened by overuse. Reused the generate path + weight + codex + the authored anchors — nothing reinvented. Rating-aware. Additive (`CONTENT.legends`, `character.legendGovernor`, `_gen.legendTier`). Suites + parse_probe green. Never touched the ErikIAm pipeline.
