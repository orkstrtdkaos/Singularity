# Results — SNG-045 (Player identity dedup — one person, one profile)

Date: 2026-07-11 · Shipped v1.8.7 · 708 smoke checks (+12) + parse_probe green · live-verified in-browser. Status: **shipped, awaiting Erik preview-legs before complete_pending_review**. Only Aevi closes.

**The "two Eriks in the picker" fix.** Identity was keyed per-device (`playerKey` minted fresh per device), so "Erik" on the phone and "Erik" on the computer became two profiles; BATCH-7 synced characters but not identity, fragmenting one person into several profiles.

## Part A — reconcile existing duplicates (`state.js`, SNG-022-style, idempotent)
- **`planPlayerDedup(profiles, currentKey)`** (pure, exported): groups profiles by `displayName` (case-insensitive); for each group of ≥2, the **current device's key wins** the canonical slot, else the profile with the **most characters** (deterministic tiebreak by key). Distinct names are never grouped.
- **`dedupePlayers()`**: applies the plan to localStorage — unions `charactersPlayed`, **reassigns each retired profile's characters** to the canonical key (updates `character.playerKey` + re-saves), **prefers a deliberately-set rating** over a default one (and preserves a protective `isMinor` flag from either side), writes a **retired→canonical redirect**, and deletes the retired profile. **Never destroys** a character or a play-style record. Runs at boot; idempotent (a second pass merges nothing).
- For Erik: `54seyk` + `s9z9u1` → ONE Erik owning all three characters. **Drizzy (`0jwfjo`) left completely alone.**

## Part B — prevent future duplicates
- **`resolvePlayerKey`** follows the redirect chain; **`getPlayerKey`/`setPlayerKey`** resolve through it, so a device whose key was retired **repoints to canonical** automatically.
- **`findProfileByName`**: entering an **existing** display name in the picker **attaches to that person** (`setPlayerKey` to the existing key) instead of minting a new per-device key. "New player" only when the name is genuinely new.

## Guardrails held
Reconcile owns the merge (safe, idempotent, additive); NEVER destroys a character or play-style; keeps a redirect for retired keys; ties to BATCH-7 identity + cross-device sync. Suites + parse_probe green. Never touched the ErikIAm pipeline.
