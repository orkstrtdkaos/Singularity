**The buildable core of the map upgrade** — the interactive/structural Layer 1 that everything else sits on. New pure module `engine/worldmap.js` + `renderMap` wiring. (Layer 2 authored base-map art = Aevi content; Layer 3 per-location generated art = a follow-on on the SNG-035 pipeline.)

## Auto-positioning (Phase 2 — the multi-area/generated-world unblock)
- **`autoMapPositions(locations)`** gives EVERY location stable coords: authored `map.x/y` kept as-is; a location authored **without** coords, or a **BATCH-9 generated** location minted mid-play, is placed **deterministically near a positioned neighbour** (id-hash angle/distance so it's stable and non-overlapping); orphans land on a hash grid. **Same input → same output** — a place never jumps between renders.
- **`coordForGenerated`** stamps a generated location with coords **on mint** (in `handleGenerateRequests`), near its parent and clear of existing pins — so a grown place appears on the map immediately.

## Iconography (the illustrated upgrade over bare circles)
- **`iconForTags(tags)`** gives each visited place a tag-derived glyph (⛩ shrine · 🏪 market · 🏛 ruin/precursor · 💧 water · ⚒ forge · 🌲 forest · ⛰ height · 🌾 farm/mill · 🏘 settlement · 🕳 cave/hollow · 🌀 churn · 🏕 waystation; ◈ fallback).
- **`terrainClass(location)`** tints the node by its **dominant disposition pole** (`poleIntensity`) — abstract/spiritual, order/mechanical, chaos/churn, life, death, light, dark each get a fill; low-charge places stay neutral. The map reads as a projection of dispositional space.

## KG overlay (Phase 1 — "see the things, not just the places")
- **`kgOverlayEntities(character, positions, npcs)`** places the codex's known **person-entities** near their home-location node (from `homeLocation`): **met** (in the npc registry) → solid; **heard-of only** (in the codex, not encountered) → dimmed/dashed — the same visual grammar as sub-places. Entities fan around a shared node so they don't stack.
- A **"Show known people"** toggle (map ⇄ map+KG) keeps the base map uncluttered; clicking an entity opens **its codex entry**.
