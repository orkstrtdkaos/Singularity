# Results — SNG-047 (Skill sidebar — group by type, show functions, adopt the combo)

Date: 2026-07-11 · Shipped v1.8.9 · 719 smoke + parse_probe green · boots clean. Status: **shipped, awaiting Erik preview-legs before complete_pending_review**. Only Aevi closes.

The abilities sidebar was a flat rank·technique·energy list with the discovered combo (✦ Resonant Sight) dangling orphaned at the bottom. Now legible and grouped.

## What shipped
1. **Grouped by type/tradition.** Owned abilities are sectioned under their power-system (harmonic / radiant / valley_craft / precursor / baseline / learned) — the **same taxonomy as the skill graph** — in **collapsible** `<details>` groups, ordered consistently.
2. **Function chips.** Each ability shows its `functions` tags (heal ✚ / shield ⛨ / strike ⚔ / reveal ◉ / conceal ◌ / bind ⛓ / move ➤ / break ✷ / ward ⬡) as small labelled chips — the player sees what an ability **does** at a glance, not only its name and technique.
3. **The orphan adopted.** Discoveries now live in their own **"Discoveries & Combinations"** group and **show their source abilities** (from the discovery's `abilityIds` / recipe `parts`), so ✦ Resonant Sight reads as **`= X + Y`** — a braid of its components, not a floating entry.

## Guardrails held
Reused ability `functions` + the discovery's recorded `abilityIds` (recipe parts) + the existing rank/energy display — nothing reinvented. **Display-only** (no mechanic change); grouping is collapsible so a big kit stays manageable; the ⓘ hover title is preserved on every row. Suites + parse_probe green. Never touched the ErikIAm pipeline.
