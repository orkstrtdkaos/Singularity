# Results — SNG-051 addendum ("▶ Run this scenario" leg-runner)

Date: 2026-07-11 · Shipped v1.8.13 · 740 smoke + parse_probe green · verified in-browser. Status: **shipped, awaiting Erik preview-leg before complete_pending_review**. Only Aevi closes.

Turns the SNG-051 passive checklist into a **one-click test-runner** — the real bottleneck-killer. Aevi authored the `force` intents in `data/preview_legs.json` (buildVersion 1.8.3, +the SNG-035 legs); CCode built the runner that resolves them.

## What shipped
- A solo leg with a **resolvable `force` intent** gets a **▶ Run this scenario** button. The runner resolves each intent against the **existing primitives** — no new mechanics:
  - **openSkillPicker / openCodex** → the character-sheet / codex screens
  - **setupSkill** `{skill, atRank}` → dev-grant the skill at that rank so the fork/rank UI is reachable
  - **setRating** `{cycle}` → `setRating` (+ adult gate) and land on Settings; the note prompts re-running to cycle R+ → G
  - **travelToDisposition** → travel to the most strongly-charged/tilted known location (reachable-first)
  - **enterPlanApt** → open a plan-apt, multi-obstacle scene (surfaces the gambit hint)
  - **timeAction** `{action}` → the sleep/rest action
  - **generateHere** `{type}` → synthesize a GM `generateRequest` (the real generate path)
  - **jumpClock** `{advanceWorldDays}` → **DEV-ONLY** shift the shared world-epoch +N days, then tick (offscreen advancement / away-digest apply)
  - **portraitBeat** → force a portrait re-mint + open the gallery
  - **grantTestState** → seed a quest-in-progress + duplicate-named items (the resolver/quest-log leg)
- **Cross-player** legs (`force: null`) and **two-character** legs (`force.intent: null` / `force.manual`) get **no button** — a "manual — …" label instead.
- **Dev-only** (the panel is `devEnabled`-gated). Intents that need an active character **guard** and route to the roster with a message. **Unknown intent → "(unsupported intent)"**, never a crash — the runner's vocabulary grows as Aevi adds intents to the JSON.
- Clock-jumps + grants are dev affordances only; `jumpClock` shifts the epoch reversibly (re-set to restore) and never touches a synced save destructively.
