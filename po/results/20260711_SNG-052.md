# Results — SNG-052 (Adult-gate checkbox persistence)

Date: 2026-07-11 · Shipped v1.8.12 · 740 smoke checks (+6) + parse_probe green · verified in-browser. Status: **shipped, awaiting Erik preview-leg before complete_pending_review**. Only Aevi closes.

**Bug fix.** The "Adult gate" checkbox unchecked on reopen even though R+ persisted — unlike the minor checkbox it was bound to nothing, read only as a one-shot authorization at save (the rating stored, the confirmation didn't).

## Fix
- **`setRating`** stamps `profile.rating.adultVerified = true` on a successful **R/R+** set; a sub-R set leaves it intact (Erik can dip to G temporarily without losing the authorization).
- The **checkbox binds** to `adultVerified` (`${profile.rating?.adultVerified ? "checked" : ""}`), so it stays checked on reopen — and R/R+ keeps working across sessions **without re-checking** (the gate is satisfied by the checkbox **OR** persisted verification).
- **`revokeAdultGate`**: unchecking the gate + saving **clears** `adultVerified` **and drops** any R/R+ ceiling back to PG-13.
- An R set with **neither** the checkbox nor a persisted verification is still **refused** — the floor + adult gate stay intact. The minor-protection floor is untouched.

## Guardrails held
Additive (`profile.rating.adultVerified`, default `false`; backfilled by `ensureRating`/`defaultRating`); minor-protection floor unaffected; no other behavior changed. Suites + parse_probe green. Never touched the ErikIAm pipeline.

## Verification
- `node tests/smoke.mjs` — **740 green** (+6): fresh profile not verified; authorizing R persists `adultVerified`; persisted verification satisfies a later set; a sub-R set keeps it; `revokeAdultGate` clears it + drops below R; R without gate-or-verification still refused.
- `node tests/parse_probe.mjs` — green.
- **Live-verified in-browser** (v1.8.12): `setRating(R+, gate)` → `adultVerified` true; reopen (`canSetRating` with the persisted flag) → ok; `revokeAdultGate` → `adultVerified` false, ceiling PG-13. Boots clean.
