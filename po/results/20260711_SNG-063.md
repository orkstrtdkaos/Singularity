# Results — SNG-063 Creation content + the ORDER fix

Date: 2026-07-11 · Shipped v1.8.20 · 765 smoke checks (+1) + parse_probe green · live-verified on a fresh port. Status: **shipped, awaiting Erik preview-legs before complete_pending_review**. Only Aevi closes. Folds into the SNG-062/059 creation build (CCode's lane).

Fixes the bug Erik found in live play: *"still the 3 starting locations… a limited set of backgrounds. Plus you can choose skills before you get through your domain choices — that's backwards."*

## The ORDER fix (the actual bug — a hard rule)
**NAME → FORM → ORIGIN → DOMAINS → ABILITIES → COMPANION**, in **both doors**. Abilities are gated by domains (SNG-055); offering them *before* domains showed skills the character can't learn and made the gates unenforceable.
- **Quick-start:** step 1 is now **name + form + origin + background + attributes** — **no abilities**. Domains come next. A **new ability step comes AFTER domains** and is **FILTERED to what the domains permit** (`domainAccess.allowed`) — the antipode and over-tier picks are never shown, grouped by people.
- **Prologue:** the intro gains an origin pick; domains still **crystallize from play**, then the **used-abilities** confirm — already the right order.

## Origins + backgrounds are content now (were hardcoded 3 / 5)
- Load **`origins.json` (27 peoples)** + **`backgrounds.json` (15)** into `CONTENT.origins`/`CONTENT.backgrounds` (in-app fallbacks if the files fail to load). Deleted the hardcoded arrays. Both render as dropdowns.

## Origin ≠ starting location (the unlock)
- **ORIGIN = which PEOPLE you're from.** Stored as `character.nativeTradition` — `homeClassOf` now prefers it, so a **native Umbral isn't cross-classed** for Umbral craft — and it **SEEDS the primary domain** on the great circle (a pole origin pre-fills the primary, which the player can Redo; folk origins stay open). Every origin's **`whyYouAreHere`** is stored and folded into the bio so the **GM has it** as grounding.
- **STARTING LOCATION = the Valley, always** (the only playable region until SNG-060). A character can be an **Umbral who came to the Valley** — native craft, Valley streets.
- **BACKGROUND = what you did**, orthogonal — **not gated by origin** (a Cogitant soldier, a Marcher healer).

## Guardrails held
Content-driven (no hardcoded origins/backgrounds); domains locked before a single ability is offered, then filtered; origin seeds native access + circle position without conflating with location; migration untouched (legacy origins fall back through `homeClassOf`; existing characters keep everything, reconcile-v5 still infers their domains). Suites + parse_probe green. Never touched the ErikIAm pipeline.
