## Phase 1 — Gambit surfacing (SNG-031) + completion-XP wire (SNG-030 remainder): SHIPPED (v1.7.6)
- **SNG-030 fixed — the completion bonus was never paid.** `finishGambit` computes the outcome and then, on a **non-abandoned** completion, awards `rules.gambit.completionBonusXp` (10) to `character.xp`, runs `applyLevelUps` so a gambit can push a level *now* (not on the next rolled action), and surfaces `+N xp` in the gambit result label. An **abandoned** gambit still pays nothing. Strategist's extra adaptation point was already wired (`adaptationPointsFor`) and is now covered by a test so it can't silently regress.
- **SNG-031 — the fully-built gambit was hidden; now it surfaces when the scene warrants it.** `isGambitApt(turn)` marks a scene Plan-apt when it has **≥3 choices** AND any of: a `plan`/`scout`/`prepare`/`investigate`/`analyze` intent tag, **≥2** ability-relevant choices, or **≥4** non-trivial choices. Routine 1–2 choice scenes never trip it.
- **Surfacing (non-nagging):** a dismissible **gambit-hint chip** sits beside the freeform input on apt scenes; the ⚙ Plan button gets `.apt` emphasis only when apt. Dismissal (`gambitHintDismissed`) is respected until the next `startScene`, so it hints once per scene, not every turn.
- Live-verified: award + level-up + label semantics unit-tested exactly as `finishGambit` runs them; apt-detection boundaries unit-tested; chip + `.apt` render paths present in shipped `app.js`.

## Phase 2 — Narrative-driven time (SNG-032): SHIPPED (v1.7.6)
- **The GM now moves the world clock with the fiction.** GM contract gains `timeOps: {hoursPassed, why}`; rule text added: *"timeOps: TIME MOVES WITH THE FICTION — narration LEADS the world clock."* `timeOps` is registered in the `salvageOps` key list so a partially-malformed turn still recovers it.
- **`applyTurn` honors the declaration, clamped:** when `timeOps.hoursPassed` is finite it **REPLACES** the beat default (clamped **0.25–72h**); otherwise it falls back to the old `beat/sceneEnd + legacy timeAdvanceHours` behavior unchanged. Sleeping advances ~8h, a quick exchange ~0.5h, a multi-day trek clamps at 72h, a floor of 0.25h prevents zero-time turns. `advanceClock` already accepted fractional hours, so minutes-scale ticks work without engine change.
- **Backward-safe:** no `timeOps` = exactly the prior clock behavior; a declared value never *reduces* below 0.25h or exceeds 72h regardless of what the GM emits (engine owns the clamp — Design law 1).
- Live-verified: replace/fallback/clamp/floor all unit-tested against the mirrored `applyTurn` rule; contract + rule + salvage-key confirmed in shipped `gm.js`.
