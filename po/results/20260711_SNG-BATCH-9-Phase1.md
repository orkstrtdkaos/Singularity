## 1a — Derived npc/location/arc schemas + a minimal validator (BUILD STEP 1, do-first)
- `schemas/npc.schema.json`, `location.schema.json`, `arc.schema.json` — DERIVED from the authored corpus so `generate()` validates against the same shape the hand-authored content has. NPC required = the 30/30-real core; location = the 14 fields all 26 authored places share; arc = the 9 all 5 greater-arcs share (`scale` enum gains `local`). The green gate caught two real corpus variations my first pass guessed wrong (`axisVector` is numeric, `encounterSeeds` items are string-or-object) — schema corrected to match reality.
- `engine/genschema.js` — a dependency-free subset validator (no build step = no ajv): type/required/properties/items/enum/additionalProperties + `missingRequired` + `defaultFor` for the repair step.
- Green gate: all 61 authored records validate; negative controls confirm the validator REJECTS missing-required / wrong-type / bad-enum.

## 1b — generate(type,context) core + promotable store (v1.7.7)
- `engine/generate.js` — the path: build a schema-constrained prompt (few-shot from the authored corpus, under the local disposition grammar from `generative_substrate.json`) → validate → auto-repair missing-required from context → minimal in-grain stub if still invalid (**NEVER halts a turn**) → **resolve-before-mint** via the SNG-019 namematch (reuse over duplicate) → stamp the promotable `_gen` envelope → persist to a per-save store + a codex mirror node. The LLM call is **injected**, so every branch is headless-testable.
- **Born promotable + SNG-041-ready:** `_gen` = `{ entityId, type, birthWeight (=birth-power), engagementScore, tier, rating, provenance{playerKey, characterId, locationId, day, worldDay, hint, why}, attentionHistory, createdDay }`. The `worldDay` slot is the SNG-041 forward-compat hook — the one-clock fills it with zero migration.
- **Wiring:** GM contract gains a lightweight `generateRequest` op (+ rule "the world grows… prefer reuse, request only genuinely new content, reactive only" + salvage key); `claude.js` routes `generate` → Sonnet; `loadContent` fetches the substrate + greater_arcs + the 3 schemas (optional — a miss disables generation, never breaks load); `app.js` **hydrates the character's grown world into the live CONTENT maps** on enterPlay (generated content is first-class + **revisitable**, indistinguishable downstream), a per-scene **governor cap** (3), and a light "✦ the world grows" note. A generation failure never halts the turn.
- Live-verified end-to-end in-browser: a location `generateRequest` mints, validates, persists, registers revisitable, connects back, carries the `_gen` envelope + codex mirror.

## 1c — Content-rating ceiling + the two floors (v1.7.8)
- **§3 ceiling (Erik-set per family profile):** `playerprofile.js` presets G/PG/PG-13/R/R+; `canSetRating` enforces all three control rules — a minor profile is hard-capped at PG-13, R/R+ require the adult gate Erik controls, and a profile **cannot raise its own ceiling**. Three consumers wired: (a) GM narration tone (a CONTENT CEILING tier-1 block naming the ceiling + floors), (b) generation ceiling (`generate()` authors to the requesting player's ceiling), (c) per-entity rating tag (`_gen.rating`). Settings UI: a rating select + minor + adult-gate controls, guards enforced on save.
- **§4 the floors (rating-INDEPENDENT, absolute, at the birth-validator):** a sexualized minor is stubbed entirely (prohibited floor — R+ NEVER unlocks it), a minor's softer romantic framing is neutralized, every minor NPC is stamped `_gen.romanceEligible=false`. Adult sexual/romantic content is NOT blocked (intensity is the ceiling's job, not the floor's).
- Live-verified through the real save handler: R+ refused without the gate, accepted with it, and a minor profile capped to PG-13 with the reason surfaced.

## 1d — Engagement governor + canon tiers (v1.7.9)
- **§2 the anti-sprawl governor:** attention (revisit/interact/fact/quest/session) keeps a grown thing real; inattention lets it go dormant — **stops propagating, never deleted (what happened happened).** Tiers fresh → established (durable personal canon) → nominated (Phase-3 promotion queue). Established never falls back from inattention; only FRESH goes dormant past the window. `effectiveWeight = birth-power + accumulated attention` (realness, recorded from birth so Phase-3 promotion is zero-rework). `livingWorldForGM` surfaces only non-dormant grown content relevant here, tier+weight tagged; the GM references it as "already real, never re-introduce as new."
- **Wiring:** revisit on travel, interact on npcUpdates, fact on codexUpdates; the LIVING WORLD block threaded into the GM context.
- Live-verified: a returned-to entity stays established + surfaces + gains weight; an ignored one goes dormant + fades from what the GM raises, but stays in the store (not deleted).
