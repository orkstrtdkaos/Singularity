# Results — SNG-066 In-game feedback with auto-captured context

Date: 2026-07-12 · v1.8.35 · npm test green · live-verified on a fresh port. Status: **shipped, awaiting Erik leg.** Only Aevi closes.

## Why
Every bug this session arrived as a screenshot + a description and cost Aevi 2–4 tool calls to reconstruct *where Erik was, what version, what state.* A feedback button that **auto-captures the context** turns a 5-minute archaeology dig into a one-tap report — every bug this session would have arrived pre-diagnosed.

## What shipped — the value is the FORENSICS
- **A persistent `⚑ Feedback` control** in the header, on every screen (creation, prologue, play, settings).
- **One-tap type:** 🐛 Bug · 💡 Idea · 🤔 **Felt off** (the third is where "not broken, just wrong" finally has a home).
- **AUTO-CAPTURE, no typing:** app version · screen · character (id / name / level / domains / origin / background) · location + region · active quests · scene setting · **last GM turn** (word-boundary-clamped) · last player action · **runtime errors since load** (`window` `error` + `unhandledrejection` listeners registered at boot) · **world-day + real timestamp**. Shown in an expandable "what will be attached" panel so the player sees exactly what they're sending.
- **Writes append-only to `po/feedback/{YYYY-MM-DD}.json`** via `pushMergedFile` (read-merge-write, SHA-retry, id-deduped) — concurrent reporters never clobber.
- **Sync off → queued in localStorage** and flushed on the next `enterPlay` when sync returns. **Never loses a report.** Confirms with the entry id.
- **Redaction:** only the named fields are captured — never the config object / PAT (verified: no credential string in the entry).

## Verification
- `npm test` + `parse_probe` + `content_ci` green; boots clean.
- **Live-verified on a fresh port:** the ⚑ control opens the modal; all three types select correctly (verified "Felt off" → the queued entry's `type` is `feel`); the context auto-captures (appVersion `1.8.35`, screen, worldDay, + character/location/lastTurn/errors when in play); submit with sync off **saves** the entry ("✓ Saved as fb-… — will send when sync is on. Nothing is lost.") and it persists in the localStorage queue with its full context; **no credential leak**; no console errors.
