# Results — SNG-070 GM corrections (the game self-heals)

Date: 2026-07-12 · v1.8.30 · npm test green · boots clean on a fresh port. Status: **shipped, awaiting Erik leg.** Only Aevi closes.

## The case
Every creation bug this session — the auto-assigned background, the un-repickable domain, abilities from the wrong domains (**Silas Weir**, primary-domain *wright*, holding not one wright ability), the desynced location, a companion never met — needed a **code fix + redeploy to correct one player's save.** That is the wrong loop. The GM sits in the world with full state access; it should be able to fix the world.

## The build — a bounded `stateOps`, REPAIR NOT WISH
New **`engine/corrections.js` `applyStateOps`** (pure, validated, never throws):
- **ALLOWED (fix what's wrong):** `correctField` (background / origin / nativeTradition / form) · `correctDomain` (re-choose a domain the game GUESSED — **does NOT grant or remove abilities**; held ones stay grandfathered, and the corrected domains gate only future learning, per SNG-055) · `removeEntity` (companion / quest / codex / npc never acquired) · `unstickQuest` (re-stage / re-status) · `reanchorLocation` (by id or name) · `fixCodexFact` · `refuse`.
- **REFUSED, engine-enforced (never advance):** `xp` · `level` · `skillPoints` · `attunement` · `health` · `energy` · `abilities` · `inventory` · `attributes`. There is simply **no grant op**, and a `correctField` on a forbidden field is rejected. Power comes from play.
- **Every correction LOGGED** to `character.corrections` (from → to, why, world-day) — append-only, reviewable, reversible. **Bounded to the asking player's own save**; never touches shared canon; cannot raise a rating ceiling or bypass minor-protection.
- **`gm.js`:** the `stateOps` reply contract + a repair-not-wish rule (recognize a correction, narrate the fix honestly in fiction, and **refuse power plainly and warmly** via `{op:"refuse"}`) + the salvage whitelist.
- **`app.js`:** dispatch in `applyTurn` with the content context for validation; a "Set right: …" aside surfaced by `renderPlay` whichever path rendered the turn.
- **`SYSTEM_SPEC §11`** updated: `stateOps`, which my round-2 flagged as unbuilt, is now built.

- **Erik leg (needs a live model):** "Tell the GM your background is wrong and should be Duelist — verify it's corrected, narrated, and logged. Then ask it for 500 XP — verify it declines, plainly and without being a prig." The engine validates/applies/logs deterministically; the GM actually recognizing the request and emitting `stateOps` is the live behavior.

## spec_boundaries
- **The correction lands via a normal GM turn** (the player types the request as their action; the GM emits `stateOps`), reusing the main loop — not a separate "Ask GM" modal. The existing read-only `gmAsk` guidance path stays as-is; corrections need to write, so they ride the turn contract.
- **The refusal is enforced by the shape of the op set** (no grant op exists) plus a forbidden-field list — the GM *cannot* advance a character even if it tries, so the guardrail does not depend on the model's goodwill.
