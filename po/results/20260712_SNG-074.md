# Results — SNG-074 Dev mode off-switch

Date: 2026-07-12 · v1.8.29 · npm test green · **verified on the LIVE production URL.** Status: **shipped, awaiting Erik leg.** Only Aevi closes. **This was the Brooklyn blocker.**

## The bug
A single `?dev=1` visit wrote a **permanent** `localStorage "singularity.dev"` flag, read on every URL forever, with **no un-enable.** Erik had no clean player view; any family member who ever opened a `?dev=1` link was permanently dev-enabled — with the clock-jump, forced encounters, and state grants (SNG-051 runner) live in their save. Not a leak — **stickiness with no way out.**

## The fix — one dev-mode source of truth: `isDevMode()`, opt-in / visible / reversible
- **`?dev=1` is SESSION-scoped** (`sessionStorage`, not `localStorage`) — a plain reload on the live URL is a **clean player view.**
- **`?dev=0` always wins OFF** and clears the session flag + any persistent opt-in.
- **The old sticky `localStorage "singularity.dev"` is MIGRATED OUT on boot** — nobody stays trapped, including anyone already stuck.
- **A visible Settings "Developer mode" toggle** — **off by default** — is the ONLY persistent opt-in (`localStorage "singularity.devPersist"`), chosen deliberately, with an explicit off.
- **A `DEV` badge in the header** whenever dev is active — no ambiguity about which view you're looking at.
- **localhost/preview auto-on stays** (a real dev environment).
- **Audit:** both historical gates (`devEnabled()`, `isDev()`) now resolve to the single `isDevMode()`; every dev surface — the 🧪 Legs panel, the test-encounters panel, and the scenario runner's clock-jump / state-grants / setRating — is behind it. `isDev()` previously treated any `?dev` (including `?dev=0`) as on — fixed.

## Verification
- `npm test` green; boots clean on a fresh port.
- **Live-verified on localhost:** normal load → DEV badge + Legs present; `?dev=0` → clean (no badge, no Legs, no test panel) + the legacy sticky flag migrates out; Settings toggle off by default; ticking + Save persists (`devPersist="1"`); unticking + Save clears it.
- **✅ Verified on the LIVE production URL** (`orkstrtdkaos.github.io`, non-localhost, fresh session): **clean player view — no DEV badge, no Legs button.** This is the exact thing Erik needs and could not previously see.
