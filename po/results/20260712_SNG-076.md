# Results — SNG-076 Authored prose is truncated mid-word

Date: 2026-07-12 · v1.8.32 · npm test green · live-verified on a fresh port. Status: **shipped, awaiting Erik leg.** Only Aevi closes.

## The bug (two, and the first is conceptual)
1. **AUTHORED content was clamped like untrusted model output.** Quest stakes/premise/objectives/outcome-narration and location `descriptionSeed` come from content packs — finite, deliberate, meant to be read. Clamping them isn't safety; it's vandalism of the thing the player reads. Erik saw stakes die at *"…and Ove…"* and the away-digest cut at *"the district acco ("*.
2. **Every clamp was a raw `slice(0,N)`** that cut mid-word — wrong even where a bound is right.

## The fix — classify each string by ORIGIN
- **AUTHORED renders IN FULL:**
  - Quest Log "Available here" **stakes** — was `slice(0,180)`, now whole (verified: 327 chars, ends *"…He has used it before."*).
  - Map **descriptionSeed** — was `slice(0,260)`, now whole.
  - Structured quest premise/stakes/objectives/outcome narration already rendered in full.
- **MODEL output clamps PROPERLY:** new **`smartClamp(text, max)`** (in the dep-free `namematch.js`) cuts on the **last word boundary** and appends a real ellipsis — never mid-word, never a dangling partial word.
  - Freeform quest note/summary: 200/240 → **600**.
  - `worldtick` away-digest + news + evolution notes: 220/200 → **600** (a stored news item no longer loses its own verb).
  - Quest world-event text (authored effect prose): 300/240 → **800** (generous safety bound only).
- **CSS audited:** no `line-clamp` / `overflow:hidden` / `text-overflow` on prose panels — nothing truncates in the layer above the string either.

## Verification
- `npm test` green — 4 SNG-076 checks: text under the bound returns whole; a 240+‑char authored string survives; an over-long model string cuts on a word boundary; the clamp never leaves a partial word.
- **Live-verified on a fresh port:** the Edge District Ledger stakes render the **full 327 chars** in the Quest Log (was cut at 180); no console errors.

## spec_boundaries
- **Model-output bounds raised to 600–800, not removed.** The clamp exists to stop a runaway model writing 10KB into a stored field; a normal sentence now survives comfortably. If Aevi wants specific per-field bounds tuned, they're single constants.
- **"Expandable … more" affordance not added** — with authored content rendering in full and model bounds raised to 600+, nothing user-facing is currently truncated, so an expander would have nothing to expand. If a future long model string hits the bound, the word-boundary ellipsis is in place; a "… more" toggle can be added then.
