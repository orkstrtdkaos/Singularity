# Results — SNG-081 The player's words are kept (the GM stops talking to itself)

Date: 2026-07-12 · v1.8.39 · npm test green · boots clean on a fresh port. Status: **shipped, awaiting Erik leg.** Only Aevi closes. **The highest-priority bug in the project.**

## The bug (definitive)
The turn record stored **only** the GM's own `sceneSummary` + `narration`. The player's input was **never kept**. So the "conversation history" fed back to the GM each turn was a **monologue of its own prose** — the player's words reached it for **exactly one turn** (`exactWords`, in the uncached tier at the very end of the prompt) and then vanished. When the GM told Erik *"no romantic overtures have appeared because none have been played,"* it wasn't refusing — from its view of history that was **true**. It had been listening to itself the whole time.

This wasn't a romance bug — it was a **total-continuity bug**: every nuance the player put in their own words (a promise, a name, a tone, a joke, a threat, a plan, a flirtation, a refusal) died after one turn. It silently violated **Design Law 4 (permanence)**: the world remembers everything — facts, codex, places, chronicle, shared canon — and forgot the player.

## The fix
1. **Store the player's half.** The turn record is now `{ player, summary, narration }` — `player` = the literal `exactWords`, else the action label (threaded from `runGM` into `applyTurn`).
2. **Render history as a DIALOGUE**, not a monologue — new pure `renderSceneHistory` (gm.js): `YOU: "let's call this hunt a date"` / `GM: <summary / full narration>`.
3. **The player's words are kept VERBATIM** (they're short and load-bearing); **only the GM's prose is clamped** (700). Ties SNG-076 — the player's own words are the last thing that should ever be truncated.
4. **Persists** via `activeScene.turns` → a scene reloaded from disk keeps the player's half.
5. **Centralized** in the single scene-history render, so **every `recentTurns` call-site inherits it** (the play path via `buildTiers`, the ask-GM path via `buildTurnContext`, emergence, quest guidance) — the spec's audit of sites 1976/2571/3220/3687 resolves to one render, not four patches.

## Verification
- `npm test` green — 6 SNG-081 checks: the player's words appear in history; kept verbatim (never truncated); only the GM's prose is clamped at 700; each beat reads `YOU: "…"` then `GM: …`; a system/party beat with no player words has no `YOU` line; legacy string turns pass through unchanged.
- Boots clean on a fresh port (the `applyTurn` signature change is in the hot path — no console errors).
- **Erik test (needs a live model):** "Say something distinctive — flirt, make a promise, name someone — then three turns later ask the GM about it. Verify it KNOWS what you said, in your words, and can quote it back." The storage + dialogue render are proven; the model actually using them is the live behaviour.

## spec_boundary
- **`player` = `exactWords` (free-typed) else the choice label.** A button-choice stores the label (the player's action); free text stores the literal words. System-driven beats (emergence acceptance, a party member's remote beat) carry no `player` and render as `GM: …` only — correct, they aren't the player's words.
