## Phase 3 — STRUCTURED QUESTS (v1.8.24) — authored quests load, and resolving one changes the world
`quests.json` was manifest-registered but had **no loader branch** (`provides.quests` = 0 refs) — the authored quests silently never loaded; and there was no quest structure, only freeform GM quests.
- **Loader branch** in state.js → `CONTENT.quests` (optional, never breaks load).
- **engine/quests.js** structured quests alongside the freeform system: `isRealQuest` (the schema's stakes-or-it's-an-errand rule), `startStructuredQuest` (idempotent), `completeQuestStage` (records the stage's `change` as findable progress + advances), and `resolveStructuredQuest` — which **APPLIES the outcome's consequences**: ALWAYS a findable chronicle record (the schema's floor), plus best-effort parse of the tagged prose for a propagating **WORLD-EVENT** (dated on the shared clock via an injected sink → a pinned established fact), disposition shifts, and NPC state. `availableStructuredQuests` (offer where the giver's present / region matches), `routesForCharacter` (the routes a character's DOMAINS open), `structuredQuestsForGM`.
- **gm.js**: a STRUCTURED QUESTS block — honor stakes + current stage, weave the domain-opened routes, never resolve a stage yourself (the engine applies it).
- **app.js**: Quest Log gains an "Available here" section (Take it on) + a structured detail view — stakes, stages with a mark-met control, domain-lit routes, and branched outcomes that resolve + show what changed.
- **12 smoke checks** + **full live walkthrough:** a valleyfolk in Millbrook sees the Edge District Ledger available; its detail shows stakes, 4 stages, **verist + lattice lit as the character's open routes**, 4 outcomes; resolving "Grael Exposed" wrote a chronicle record + established fact + world-event + verist disposition +2 + 30 xp. ## spec_boundaries
- **Quest consequences are authored as tagged prose**, so mechanical application is best-effort: the WORLD-EVENT + "X disposition … raised/lowered" forms parse cleanly, but an elliptical form like "Veilwright: lowered" (no adjacent "disposition" keyword) is NOT parsed into a delta — though its full text is always preserved in the findable chronicle/fact record. If Aevi wants every disposition shift mechanically live, the cleanest path is a small machine-readable `effects[]` alongside the prose per outcome. Flagged, not silently decided.
- **Quest stage conditions are engine-TESTABLE in principle but advanced manually** (a "mark this stage met" control) — auto-detecting "spoke with Fendt / obtained the log" from live fiction is a GM-integration follow-on, not built this batch. The structure + resolution + durable consequences are all live.
- **Content CI is a local/`npm test` gate**, not a GitHub Action (no CI workflow exists in the repo). If Aevi/Erik want it enforced on push, a one-file `.github/workflows` running `npm test` would do it — noted, not added unprompted.
