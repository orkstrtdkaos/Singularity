# Results — SNG-088 (the gambit builder, fixed four ways + reorder + auto-fill from conversation)

Date: 2026-07-13 · v1.8.46 · npm test green (smoke + parse_probe + content_ci). Status: **shipped, complete_pending_review.** Only Aevi closes.

Erik: *"words are still getting cut off… I tried to have the GM set up a gambit, but it ran it as one big input… now I'm trying to set it up properly, but it thinks I failed the first time already."* The mechanic the game is best at was unusable. All four spec parts fixed, plus two Erik-requested additions (reorder, auto-fill from the conversation).

## A — SNG-076 missed the gmAdvice slice (and SNG-076 does NOT close yet)
`app.js:4076` hard-cut the GM's plan-advice at `slice(0,400)` — mid-word, no ellipsis. Now `smartClamp(text, 600)` (the SNG-076 word-boundary + real-ellipsis clamp) with the full text kept and a **more/less** expander.
- **Verified:** a 637-char advice string → clamped to 600, the cut lands on a **word boundary**, ends with a real `…`, and the **more** toggle appears (600 < 637). No mid-word butchering.
- Per §21 (close on the symptom): **SNG-076 stays open until Erik confirms no text is cut anywhere.**

## B — the GM can now OPEN a gambit (it had no mechanism, so it ran the plan as one action)
New **`gambitOps`** turn op. `gm.js` contract: when the player says they want to PLAN a multi-step approach, the GM emits `{goal, steps:[{text,fallback}]}` to **OPEN the builder pre-filled** — never resolve the plan as a single action. It carries the recovered plan-level rule verbatim from `gambit_design.json`: **HOLD THE PLAN-LEVEL VIEW AGAINST THE PUZZLE'S STEPWISE PULL** — a gambit is declared as a whole and resolved as a whole. `applyTurn` validates the proposal (goal + ≥1 step, capped at maxSteps) into `pendingGambitProposal`; `renderPlay` opens the builder editable (Design Law 1 — GM proposes, player edits + commits, engine runs). `gambitOps` added to `salvageOps` so a truncated reply still recovers the plan.
- **Verified at the parse + logic level:** `parseLooseJSON` cleanly parses a `gambitOps` turn payload (with the op present); the `applyTurn` validation + `renderPlay` guard are wired and reviewed. **Full in-play confirmation is the Erik test** — the preview harness's stubbed GM-turn response truncated under test (tripping the salvage path), which blocked a clean end-to-end drive of this one path; it is not reproducible against a complete response.

## C — abandon / reset (a failed run poisoned the next)
New `abandonGambit()` resets the draft to a **fresh blank** (no carried goal, steps, assessment, advice, actions), and a **↺ Start over** control sits in the builder. Abandoning costs nothing and pays nothing (per canon, only a *completed* gambit pays the bonus).
- **Verified live:** a filled 2-step plan → **Start over** → collapses to one blank step, goal cleared.

## D — full-width inputs
Goal, each step, and each fallback are now full-width prose fields (`width:100%; box-sizing:border-box`) instead of clipped boxes.
- **Verified live:** goal + step inputs fill their container.

## Erik-requested additions
**Reorder steps** — **▲ up / ▼ down** buttons per step; a move invalidates the assessment (ordering is meaningful — fewer steps = less slack, and sequencing changes the outcome). Edits are read before the swap so nothing is lost.
- **Verified live:** STEP-A / STEP-B → ▲ on step 2 → **STEP-B / STEP-A**, edited text preserved.

**Auto-fill from the conversation** — if the player has been *talking a plan through with the GM*, opening the builder pulls the goal + ordered steps (+ any discussed fallbacks) out of the recent dialogue instead of making them retype it. New `gm.js` **`extractGambit({recentTurns})`** (same shape as gambitOps), a **✦ Fill from our conversation** button, and **auto-fill on open** when a discussion is present (`hasPlanDiscussion()` — ≥2 recent turns, ≥300 chars). Extracted, then fully editable and reorderable (Law 1). A typed goal survives an empty extraction.
- **Verified live end-to-end:** two plan-discussion turns with the GM → opened the builder → it auto-detected the discussion and populated **goal** ("take the falsified purity ledger out of the array office") + **3 ordered steps** (scout / force the lock / carry it out) + **2 fallbacks** (alcove / behind the clerk), with the status "✦ Pulled from your conversation — edit anything, reorder as you like, then Assess and Run."
