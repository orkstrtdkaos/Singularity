# Results — SNG-089 Phase 1 (the notFor LAW + harm rungs + the Accords engine ungate)

Date: 2026-07-13 · v1.8.48 · npm test green (smoke + parse_probe + content_ci). Status: **Phase 1 shipped, complete_pending_review.** Only Aevi closes. Phase 2 flagged below.

The defect this class fixes: `notFor` clauses were authored as poetry that forbade the *need itself* — palework tagged FIGHT with prose saying it *"does not force the living to die"*, so a necromancer could not kill a boar. The GM reads the prose, so the prose won.

## Shipped (Phase 1)
**1. THE notFor LAW — a CI check (headline).** `tests/content_ci.mjs` now flags any ability whose `challengeTypes` include FIGHT while its `notFor` **forbids the need** (harm/fight/hurt/injure/damage · "on/against the living" · "force … to die" · "help you escape/flee") — never the *how*. Patterns are **conservative** so a legal **degree-cap** ("does not slay", "does not smite") can never false-fail the build and block ships. Also validates `harmRung` is a real rung. **Verified it discriminates:** catches the boar bug ("does not force the living to die"), "cannot be used to fight", "cannot be used on living things"; ignores `the_grey_hand`'s legal "does not slay" and `death_ward`'s "harming the living" list. **Current content passes** — Aevi's palework/wither corrections hold, and the law now guards them going forward.

**2. Harm rungs → the GM.** `abilitiesForGM` emits a `HARM:` line per owned ability from a new `harmRungGloss` (`lethal | damaging | incapacitating | none`). *"Can fight" ≠ "can harm."* An **incapacitating** craft STOPS a threat without wounding it; a **`none`** craft (Stillhold peace-working) wounds nothing and the GM is told never to invent a wound from it (and that a foe who can't be turned must run, borrow, or take up a weapon). Feeds the GM so it narrates the idiom correctly instead of defaulting to a generic cut.

**3. The Accords — engine ungate.** `the_accords.json` loads into `CONTENT.accords`; each signatory ability is tagged `accord`. The learn-gate (`learnAbility` · `domainGateFor` · `domainVerdict`) **bypasses the origin + domain gate** for an Accord craft — anyone may take it, still for 1 point (freely *accessed*, not free to *learn*). **Verified:** `staunch` (a Rootkin craft) returns `{allowed:true, penalty:1, band:"accord"}` for a Cogitant character — the ungate works.

**4. The narrative weapon path — confirmed.** `resolveAction` requires no ability; a FIGHT action with `abilityId: null` resolves on attributes + subAttribute + equipment/companion bonuses. So a Cogitant duelist and a Marcher both fight with steel; the Marcher is simply better at it. **Craft makes you DISTINCT, not ABLE — no tradition is ever helpless.** (No code change needed; verified the path exists.)
