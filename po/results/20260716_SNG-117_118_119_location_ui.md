## SNG-117 — The known world is navigable (the load-bearing one)
Two symptoms, one root cause: the engine could only travel to / name places with a resolvable record.
- **A known-places layer, wider than visited** — `engine/worldmap.js isPlaceKnown(character, locId, locations)`: a place is KNOWN when you stand there, have visited, it's **adjacent** (one travel away via `connections`), or the fiction named it / you're en route (`character.knownPlaces`). The map (`renderMap`) and both map headers now name any **known** place; `?` is reserved for the genuinely unheard-of. `travelTo`/`moveTo` feed `knownPlaces`.
- **A named-but-unrecorded destination becomes real** — `mintTransitLocation(moveRef)`: an unresolvable `moveTo` ("the pass") mints a lightweight, **idempotent** (keyed off the normalized name — Q1) location adjacent to where you left, with a stable id + map coord, persisted on the save (`character.generated.location`, revived by `hydrateGeneratedIntoContent`). No model call — deterministic; the world-gen path can flesh it out later.
- **The header follows the fiction even on a soft exit** — the `moveTo` block resolves *or mints*, so `currentLocationId` never stays asserting a place the fiction has left. (The narration-only exit with **zero** `moveTo` — inferring a destination from prose — is the remaining boundary; the concrete `moveTo:"the pass"` case Erik hit is fully fixed.)
- **ROUND-2:** Q1 — the direct mint dedups by normalized id (`gen-<slug>`), minting once. Q3 — adjacency **is** in location records (`connections`), so "adjacent = known" is computed live. Q2 (soft-exit-without-moveTo signal) — noted as the boundary above.

## SNG-118 — Play-style as tappable chips
The prose wall is now `aptitudeChips()`: each held aptitude is a tight colored pill — tinted by **axis** (earned / amorous / inverse), **dimmed + ⌁** when FADING (SNG-113's "loss is legible"), **border-marked** for a lineage grant. Tap reuses the one popover surface (`showPopoverText`, via a delegated `[data-aptchip]` handler) to show effect + description + mods. Replaces the concat at both render sites (character screen + play sidebar); empty state is graceful. Scales with the 26-aptitude roster.

## SNG-119 — Standing folded into location headers + title
- `engine/npcs.js knownPeopleAt(character, locId, {locations, npcs})` — scopes bonds to a place (first-met/last-seen there, or the authored NPC's `homeLocation`/`communityId` matches — Q1: those fields exist). Pure.
- **Current-location title** gets a tappable standing chip ("Millbrook · *trusted*", read where you already look for where-you-are).
- The detached **"Standing here" sidebar** folds into a place-scoped **"standing & who's here"** section (community standing + the people you'd find there, e.g. Pell as "partner · devoted"), and each **map-detail location header** now carries its standing band + the people you know there.
